# OpenClaw 内容优化执行清单（SEO + 实操价值）

更新时间：2026-03-12
状态说明：`[x] 已执行`，`[ ] 待执行`

## 目标

1. 把“空泛介绍”改成“可直接操作”的内容表达。
2. 强化高频搜索词覆盖：安装、使用、技能、渠道、排障、成本。
3. 维持静态 SEO 架构（不做动态语言切换，不做客户端渲染 SEO）。

## 优先级执行列表

### P0（最高优先级）

- [x] 调研并收敛内容来源（官方优先 + 社区补充）
  - 已完成：官方文档、GitHub、知乎、B站关键词方向收敛。
  - 输出：安装/技能/渠道/排障/成本五大主题词簇。
- [x] 首页首屏强化“OpenClaw 免费安装”意图词
  - 动作：H1/H2、首屏导语、CTA 文案改成“安装命令 + 结果承诺”。
  - 验收：首屏出现“免费安装/安装教程/安装命令”自然表达，且不堆词。
- [x] 首页增加“5 分钟快速验证”微内容模块
  - 动作：展示最小命令链路（安装 -> doctor -> status）。
  - 验收：用户不离开首页即可完成第一步自检。

### P1（高优先级）

- [x] 安装专题页关键词与段落改写
  - 动作：覆盖“OpenClaw 安装教程 / 免费安装 / Docker 安装”变体。
  - 验收：每节包含命令、常见错误、验收命令。
- [x] Skills 专题补齐“怎么装 skill / 装哪些 skill”
  - 动作：增加 ClawHub 安装、本地 skills 目录、优先级规则。
  - 验收：给出最少 3 条可直接复制的命令或配置片段。
- [x] 渠道接入专题强化“消息不回复”排查路径
  - 动作：飞书/钉钉/Telegram 统一排查树。
  - 验收：至少有“鉴权 -> 触发规则 -> 会话权限 -> 日志定位”四步。

### P2（中优先级）

- [x] 排障专题标准化“先恢复、再定位、后复盘”
  - 动作：按故障处置顺序重写内容，不讲概念空话。
  - 验收：每类问题有“首条检查命令 + 回滚动作 + 根因记录模板”。
- [x] 成本专题补齐“提供商与模型分层”
  - 动作：增加主备模型、预算阈值、降级策略示例。
  - 验收：能直接指导用户设置日预算/月预算与告警阈值。
- [ ] 全站 metadata 与结构化数据查漏补缺
  - 动作：校准 title/description/keywords，补强 FAQ/Article schema。
  - 验收：中英文页面各自独立 canonical/hreflang，无动态切换依赖。

## 执行节奏（一步一步）

1. 先做 P0 首页（关键词 + 可执行命令）。
2. 再做 P1 安装与 skills（搜索流量最大入口）。
3. 再做 P1 渠道与排障（解决“消息不回/线上故障”）。
4. 最后做 P2 成本与结构化数据收尾。
